# Integración ERP: Odoo 17 / 18

> **ERP:** Odoo Community / Enterprise 17.0 – 18.0  
> **Proveedor:** Odoo S.A. — https://www.odoo.com  
> **Capa de integración:** JSON-RPC External API (`/web/dataset/call_kw`) o REST API de Odoo 17 (`/api/`)  
> **Módulos requeridos:** `base`, `sale_management`, `purchase`, `stock`, `account`, `l10n_ve` (localización venezolana)

> 🇬🇧 [English version](../../09-erp-mapping/01-odoo-17-18.md)

---

## Referencia de Endpoints API

| Operación | Protocolo | Endpoint | Notas |
|---|---|---|---|
| Autenticación | JSON-RPC | `POST /web/session/authenticate` | Devuelve `uid`; requerido antes de todas las llamadas |
| CRUD de modelos | JSON-RPC | `POST /web/dataset/call_kw` | Body: `{model, method, args, kwargs}` |
| Búsqueda REST | REST (v17+) | `GET /api/{model}?domain=[...]` | Requiere token Bearer (OAuth2) |
| Creación REST | REST (v17+) | `POST /api/{model}` | Body JSON |
| Actualización REST | REST (v17+) | `PATCH /api/{model}/{id}` | Actualización parcial soportada |

**Ejemplo de autenticación (JSON-RPC):**
```
POST /web/session/authenticate
Content-Type: application/json

{
  "jsonrpc": "2.0", "method": "call", "id": 1,
  "params": {
    "db": "fidex_prod",
    "login": "adapter@lab-leti.com",
    "password": "••••••"
  }
}
```

## 3. Orden → `sale.order` / `purchase.order`

**Mapeo de `doc_type`:**

## Notas Importantes

- **Módulo `l10n_ve`**: La localización venezolana oficial agrega validación del formato RIF, `l10n_ve_control_number`, libros de IVA y campos de retención. Instalar antes de desplegar cualquier adaptador FideX. Versiones comunitarias en https://github.com/odoo/l10n-venezuela.
- **Campos `x_*` personalizados**: Agregar desde Configuración → Técnico → Campos, o mediante un módulo personalizado que extienda los modelos.
- **Multi-moneda**: Configurar Bs (VES) como moneda secundaria de la empresa. Usar `x_bcv_rate` para sobrescribir la tasa automática cuando la tasa BCV difiere del tipo de cambio en tiempo real.
- **Bloqueo SICM**: Implementar como un decorador `@api.constrains` en `sale.order` que verifique `partner_id.x_sicm_status == 'VALID'` y `partner_id.x_sicm_expiry > fields.Date.today()`.
