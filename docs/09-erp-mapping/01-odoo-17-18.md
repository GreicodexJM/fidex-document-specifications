# ERP Mapping: Odoo 17 / 18

> **ERP:** Odoo Community / Enterprise 17.0 – 18.0  
> **Vendor:** Odoo S.A. — https://www.odoo.com  
> **Integration layer:** JSON-RPC External API (`/web/dataset/call_kw`) or Odoo 17 REST API (`/api/`)  
> **Required modules:** `base`, `sale_management`, `purchase`, `stock`, `account`, `l10n_ve` (Venezuelan localization)

---

## API Endpoint Reference

| Operation | Protocol | Endpoint | Notes |
|---|---|---|---|
| Authenticate | JSON-RPC | `POST /web/session/authenticate` | Returns `uid`; required before all calls |
| Model CRUD | JSON-RPC | `POST /web/dataset/call_kw` | Body: `{model, method, args, kwargs}` |
| REST search | REST (v17+) | `GET /api/{model}?domain=[...]` | Requires Bearer token (OAuth2) |
| REST create | REST (v17+) | `POST /api/{model}` | JSON body |
| REST update | REST (v17+) | `PATCH /api/{model}/{id}` | Partial update supported |
| File upload | JSON-RPC | `POST /web/binary/upload` | For attachments |

**Authentication example (JSON-RPC):**
```
POST /web/session/authenticate
Content-Type: application/json

{
  "jsonrpc": "2.0", "method": "call", "id": 1,
  "params": {
    "db": "fidex_prod",
    "login": "adapter@lab-leti.com",
    "password": "••••••"
  }
}
```

## 1. Customer Master → `res.partner`

**Upsert strategy:** search by `vat` (RIF); update if found, create otherwise.

| FideX field | Type | Odoo field | Model | Notes |
|---|---|---|---|---|
| `customer_details.name` | string | `name` | `res.partner` | Legal name |
| `customer_details.commercial_name` | string | `display_name` / `company_name` | `res.partner` | Trading name |

| `customer_details.gln` | string | `ref` | `res.partner` | Or custom `x_gln` field |

| `contact_info.email` | string | `email` | `res.partner` | |
| `contact_info.phone` | string | `phone` | `res.partner` | E.164 format |

**API call (JSON-RPC):**
```

  "model": "res.partner",
  "method": "create",
```

## 3. Order → `sale.order` / `purchase.order`

**doc_type mapping:**
- `PURCHASE_ORDER` (sent by buyer) → `purchase.order` at buyer's Odoo  
- `PURCHASE_ORDER` (received by seller) → `sale.order` at seller's Odoo  
- `QUOTE` → `sale.order` (state = `draft`)  
- `ORDER_CONFIRMED` → confirm existing `sale.order` via `action_confirm`

| `document_info.dates.created_at` | datetime | `date_order` | `sale.order` | |
| `document_info.dates.delivery_requested` | datetime | `commitment_date` | `sale.order` | |
| `parties.seller.rif` | string | `partner_id` | `sale.order` | Resolve to `res.partner` by `vat` |
| `parties.buyer.rif` | string | `partner_id` | `purchase.order` | |
| `parties.ship_to.rif` | string | `partner_shipping_id` | `sale.order` | |
| `payment_terms.term_type` | enum | `payment_term_id` | `sale.order` | |

## Key Notes

- **`l10n_ve` module**: The official Venezuelan localization adds `vat` format validation, `l10n_ve_control_number`, IVA books, and retention fields. Install it before deploying any FideX adapter. Community versions at https://github.com/odoo/l10n-venezuela.
- **Custom `x_*` fields**: Add via Settings → Technical → Fields, or via a custom module that extends the models above.
- **Multi-currency**: Set the Bs (VES) currency as the company's secondary currency. Use `x_bcv_rate` to override the automatic currency rate when the BCV rate differs from the live exchange.
- **SICM hard-block**: Implement as a `@api.constrains` override on `sale.order` that checks `partner_id.x_sicm_status == 'VALID'` and `partner_id.x_sicm_expiry > fields.Date.today()`.
